# EJERCICIO 1 — Preparación y diseño del portafolio

**Lección asociada:** Portafolio de productos

---

## Contexto

Antes de construir un portafolio, es fundamental **planificar su estructura y contenido**. Un portafolio improvisado transmite desorden. Un portafolio planificado transmite profesionalismo.

---

## Consigna

Prepara el **diseño y contenido** de tu portafolio profesional completando los siguientes pasos:

### Paso 1 — Definir tu marca personal

Redacta los siguientes elementos en un documento (puede ser Google Docs, Notion o un archivo `.md`):

| Elemento                        | Lo que debes preparar                                                                                  |
| ------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Nombre profesional**          | Tu nombre completo tal como aparecerá en el portafolio                                                |
| **Título profesional**          | Una frase clara con palabras clave de tu especialidad (ej: "Desarrollador Junior · Python · Django")  |
| **Descripción breve (bio)**     | 3-4 oraciones que expliquen quién eres, qué haces y qué te motiva                                    |
| **Datos de contacto**           | Correo profesional, LinkedIn y/o GitHub                                                                |

---

### Paso 2 — Seleccionar tus proyectos

Elige **mínimo 3 proyectos** desarrollados durante el curso que mostrarás en tu portafolio. Para cada uno, completa la siguiente ficha:

```
FICHA DEL PROYECTO
──────────────────

Nombre del proyecto: ___________________________________
Tecnologías usadas:  ___________________________________
Descripción breve:   ___________________________________
                     ___________________________________
Desafío principal:   ___________________________________
Solución aplicada:   ___________________________________
Resultado obtenido:  ___________________________________
Enlace (si existe):  ___________________________________
```

> 💡 **Consejo:** Elige proyectos variados que demuestren diferentes habilidades. Un proyecto de backend, uno de frontend y uno integrador, por ejemplo, muestran versatilidad.

---

### Paso 3 — Diseñar la estructura

Dibuja un **esquema o mapa** (puede ser en papel, Figma, Canva o un documento) que muestre cómo se organizará tu portafolio. Asegúrate de incluir las **4 secciones esenciales**:

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  01  INICIO         → Bienvenida + propuesta de valor   │
│  02  SOBRE MÍ       → Tu historia + foto/avatar         │
│  03  PROYECTOS       → Tus 3+ mejores trabajos          │
│  04  CONTACTO        → Formulario o datos directos      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

---

### Entregable del Ejercicio 1

Un **documento** (digital o en papel) con:

- ✅ Tu marca personal definida (nombre, título, bio, contacto)
- ✅ Fichas completadas de al menos 3 proyectos
- ✅ Esquema de la estructura de tu portafolio

# EJERCICIO 2 — Implementación y publicación del portafolio

**Lección asociada:** Herramientas para la implementación de un portafolio

---

## Contexto

Con el diseño preparado, es momento de **llevarlo a la realidad**. Vas a implementar tu portafolio utilizando una de las plataformas vistas en clase y publicarlo para que sea accesible públicamente.

> 📊 **Dato:** Aproximadamente el 90% de los responsables de contratación del sector tecnológico consultan repositorios públicos de GitHub como parte de su proceso de selección.
>
> _Fuente: Moldstud Research, "GitHub and Developer Careers" (2024)_

---

## Consigna

### Paso 1 — Elegir tu plataforma

Selecciona la plataforma que **mejor se adapte a tu proyecto y perfil**:

| Plataforma         | Ideal para                                        | Resultado                                         |
| ------------------ | ------------------------------------------------- | ------------------------------------------------- |
| **GitHub Pages**   | Proyectos de código (HTML, CSS, JS, Python)       | Sitio web público con URL propia                  |
| **Behance**        | Diseño UX/UI, productos visuales                  | Proyecto publicado en tu perfil de Behance        |
| **YouTube**        | Demos, walkthroughs, presentaciones en video      | Video público con enlace compartible              |
| **Notion**         | Documentación estructurada y portafolio libre     | Página pública con enlace compartible             |

---

### Paso 2 — Implementar el portafolio

Sigue las guías paso a paso de la teoría del módulo según la plataforma elegida.

**Si eliges GitHub Pages:**

```
Paso a paso resumido:
────────────────────

1. Crea un repositorio en GitHub (ej: mi-portafolio)
2. Sube tus archivos (HTML, CSS, JS)
3. Ve a Settings → Pages
4. Selecciona la rama main y la carpeta raíz
5. Espera a que se genere el enlace público
6. Verifica que el sitio se vea correctamente en el navegador
```

**Si eliges Behance:**

```
Paso a paso resumido:
────────────────────

1. Inicia sesión en behance.net
2. Crea un nuevo proyecto
3. Sube imágenes, prototipos o capturas de tu trabajo
4. Incluye descripción narrativa: contexto, proceso, resultado
5. Añade etiquetas relevantes
6. Publica y obtén el enlace
```

**Si eliges YouTube:**

```
Paso a paso resumido:
────────────────────

1. Prepara un guion breve de tu demo
2. Graba tu pantalla mostrando el proyecto en acción
3. Sube el video a YouTube Studio
4. Agrega título descriptivo, descripción y etiquetas
5. Define la visibilidad (pública o no listada)
6. Publica y obtén el enlace
```

---

### Paso 3 — Aplicar buenas prácticas de publicación

Revisa que tu portafolio publicado cumpla con lo siguiente:

```
Checklist de publicación:
─────────────────────────

□ El contenido es claro y sin errores ortográficos
□ Los proyectos tienen descripciones y contexto
□ Los enlaces funcionan correctamente
□ Se puede acceder desde un dispositivo móvil
□ Tu información de contacto está visible
□ Tu foto o avatar profesional está incluido
```

---

### Paso 4 — Vincular con tu perfil profesional

Agrega la URL de tu portafolio publicado a **al menos uno** de los siguientes lugares:

- Tu perfil de LinkedIn (sección Destacados, Acerca de, o Información de contacto)
- Tu CV o documento de presentación profesional
- Tu perfil de GitHub (README del perfil o bio)

---

### Entregable del Ejercicio 2

- ✅ Portafolio publicado y accesible públicamente (enlace funcional)
- ✅ Al menos 3 proyectos visibles con descripción
- ✅ Información profesional y de contacto incluida
- ✅ Enlace vinculado a tu perfil de LinkedIn, CV o GitHub

# EJERCICIO 3 — Revisión, feedback y cierre

**Lección asociada:** Finalización del proyecto

---

## Contexto

Un producto que no pasa por revisión y feedback **no es un producto terminado**. En esta última etapa vas a someter tu portafolio a una revisión crítica, incorporar mejoras y preparar tu entrega final.

> 📊 **Dato:** Las organizaciones que adoptan prácticas de retroalimentación continua experimentan un aumento del 21% en la satisfacción del cliente.
>
> _Fuente: Psico-Smart, "Feedback Continuo en Organizaciones" (2024)_

---

## Consigna

### Paso 1 — Revisión personal

Revisa tu portafolio publicado usando la siguiente checklist de calidad:

| Criterio                                     | ✅ / ❌ | Observaciones                     |
| -------------------------------------------- | ------- | --------------------------------- |
| Todas las funcionalidades son operativas     |         |                                   |
| Los proyectos están claramente descritos     |         |                                   |
| El diseño es limpio y profesional            |         |                                   |
| La navegación es intuitiva                   |         |                                   |
| Es accesible desde dispositivos móviles      |         |                                   |
| No hay errores ortográficos ni de formato    |         |                                   |
| Los enlaces externos funcionan               |         |                                   |
| La información de contacto es correcta       |         |                                   |

---

### Paso 2 — Solicitar feedback a un/a compañero/a

Intercambia el enlace de tu portafolio con **al menos un/a compañero/a** de clase. Cada uno debe responder las siguientes preguntas sobre el portafolio del otro:

```
Formulario de feedback entre pares:
───────────────────────────────────

1. ¿Qué fue lo primero que te llamó la atención?
   → _______________________________________________

2. ¿Quedó claro qué hace esta persona y qué ofrece?
   → _______________________________________________

3. ¿Qué proyecto te pareció más interesante? ¿Por qué?
   → _______________________________________________

4. ¿Encontraste algún error, enlace roto o texto confuso?
   → _______________________________________________

5. ¿Qué mejorarías o agregarías?
   → _______________________________________________

6. Del 1 al 10, ¿qué nota le pondrías a la experiencia general? ___
```

---

### Paso 3 — Incorporar mejoras

Con base en el feedback recibido, realiza **al menos 2 ajustes concretos** en tu portafolio. Documenta brevemente qué cambiaste y por qué:

```
REGISTRO DE MEJORAS
───────────────────

Mejora 1: _______________________________________________
Razón:    _______________________________________________

Mejora 2: _______________________________________________
Razón:    _______________________________________________

Mejora 3 (opcional): ____________________________________
Razón:    _______________________________________________
```

---

### Paso 4 — Presentación breve

Prepara una **presentación de máximo 5 minutos** donde expliques:

| Punto                          | Pregunta guía                                                   |
| ------------------------------ | --------------------------------------------------------------- |
| **Tu portafolio**              | ¿Qué plataforma elegiste y por qué?                           |
| **Tus proyectos**              | ¿Cuál es tu proyecto destacado y qué problema resuelve?        |
| **Feedback recibido**          | ¿Qué te dijeron y cómo lo incorporaste?                       |
| **Aprendizajes**               | ¿Qué aprendiste durante el proceso de construcción?            |

---

### Entregable del Ejercicio 3

- ✅ Checklist de revisión completada
- ✅ Feedback de al menos un/a compañero/a documentado
- ✅ Registro de al menos 2 mejoras aplicadas
- ✅ Presentación breve preparada
