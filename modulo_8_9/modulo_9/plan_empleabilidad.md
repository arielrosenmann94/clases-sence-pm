# 📅 Semana 1 — Construir la Base

---

## Día 1-2: Crear o actualizar el Currículum Vitae

### Paso 1 — Elegir formato

- Usá un formato limpio, sin columnas, sin tablas, sin diseño gráfico complejo
- Formato **PDF** con tipografía estándar (Arial, Calibri, Inter)
- Máximo **2 páginas**

### Paso 2 — Redactar cada sección

Seguí este orden exacto:

```
┌──────────────────────────────────────────────────────────────────┐
│                ESTRUCTURA DEL CV OPTIMIZADO                      │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  📌 ENCABEZADO                                                   │
│     Nombre · Título del rol · Email · GitHub · LinkedIn          │
│     Ciudad + disponibilidad remota                               │
│                                                                  │
│  📝 PERFIL PROFESIONAL (3-5 líneas)                              │
│     Lo que aportas, no lo que eres                               │
│     ❌ "Soy proactivo con ganas de aprender"                     │
│     ✅ "Desarrollador backend con experiencia en Django          │
│        y PostgreSQL. Foco en código limpio y mantenible."        │
│                                                                  │
│  💻 TECH STACK                                                   │
│     Organizado por categoría:                                    │
│     Lenguajes · Frameworks · Bases de datos · DevOps             │
│                                                                  │
│  💼 EXPERIENCIA LABORAL                                          │
│     Empresa · Cargo · Período                                    │
│     Contexto del equipo + Herramientas + Logros con métricas     │
│                                                                  │
│  🔧 PROYECTOS PERSONALES                                         │
│     Nombre · Objetivo · Tech · Resultado · Link al repo          │
│                                                                  │
│  🎓 EDUCACIÓN Y CERTIFICACIONES                                  │
│     Curso · Institución · Año                                    │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### Paso 3 — Verificar antes de guardar

- [ ] ¿El título del CV coincide con el rol que busco?
- [ ] ¿Las palabras clave del sector aparecen en el CV? (Python, Django, API, etc.)
- [ ] ¿Hay al menos 3 logros cuantificados? (números, porcentajes, tiempos)
- [ ] ¿El GitHub está visible y actualizado?
- [ ] ¿Revisé la ortografía?
- [ ] ¿El archivo está en PDF?

> ⚠️ **El 75% de los CVs son descartados por un algoritmo (ATS) sin que un humano los lea.** Las palabras clave son tu pasaporte. _(Jobscan, 2025)_

---

## Día 3-4: Crear o Optimizar tu Perfil de LinkedIn

### Checklist de perfil completo

| #   | Elemento                | Acción                                                                  |
| :-- | :---------------------- | :---------------------------------------------------------------------- |
| 1   | **Foto de perfil**      | Fondo neutro, rostro visible, expresión profesional                     |
| 2   | **Banner**              | Imagen con tu stack o rol. Hacelo en Canva gratis                       |
| 3   | **Titular (Headline)**  | "Backend Developer · Python · Django · Open to Work"                    |
| 4   | **Resumen (Acerca de)** | 200-300 palabras en primera persona + palabras clave + llamado a acción |
| 5   | **Experiencia**         | Misma lógica del CV: logros cuantificados + tecnologías                 |
| 6   | **Aptitudes (Skills)**  | Agregar 15-50 aptitudes relevantes, las 3 más importantes primero       |
| 7   | **Proyectos**           | Vincular repos de GitHub o demos                                        |
| 8   | **Educación**           | Incluir curso SENCE y certificaciones                                   |
| 9   | **Open to Work**        | Activar el marco verde + configurar tipo de rol y modalidad             |

### Conexiones iniciales

- [ ] Conectar con al menos 20 personas de la industria TI
- [ ] Seguir 10 empresas donde te gustaría trabajar
- [ ] Unirte a 3 grupos de LinkedIn relacionados con tu stack

> 💡 **Los perfiles con "Open to Work" activado reciben 2,5 veces más mensajes de reclutadores.** _(LinkedIn, 2025)_

---

## Día 5: Preparar tu GitHub

- [ ] Verificar que el perfil tenga nombre completo y foto
- [ ] Crear un README.md de perfil (github.com/tuusuario/tuusuario)
- [ ] Verificar que tus mejores proyectos tengan README completo
- [ ] Hacer al menos 1 commit (demuestra actividad reciente)
- [ ] Pinear los 3-4 mejores repositorios

```
📄 Estructura mínima de un buen README:

  # Nombre del Proyecto
  Descripción breve de qué hace y para qué sirve.

  ## Tech Stack
  Python, Django, PostgreSQL, Bootstrap

  ## Cómo correrlo
  1. Clonar el repo
  2. Instalar dependencias
  3. Crear base de datos
  4. Ejecutar servidor

  ## Capturas de pantalla
  (al menos 1 imagen del proyecto funcionando)
```

> ⚠️ **El 68% de los reclutadores TI verifican el GitHub antes de la primera entrevista.** _(Stack Overflow Developer Survey, 2025)_

# 📅 Semana 2 — Publicar Contenido y Postular

---

## Publicaciones en LinkedIn: la estrategia del 1%

> **Solo el 1% de los usuarios de LinkedIn crea contenido, pero ese 1% genera 9.000 millones de impresiones semanales.** _(Forbes / LinkedIn, 2025)_

### Calendario de publicación semanal

| Día        | Tipo de publicación                         | Ejemplo                                                                   |
| :--------- | :------------------------------------------ | :------------------------------------------------------------------------ |
| **Lunes**  | 📚 Lo que aprendí esta semana               | "Esta semana entendí cómo funciona select_related en Django y por qué..." |
| **Jueves** | 🔧 Un error que resolví o herramienta nueva | "Me trabé con una migración en Django. Así lo resolví paso a paso..."     |

### Reglas de oro para publicar

1. **Escribe en primera persona** — la gente conecta con historias, no con tutoriales
2. **Sé breve** — entre 100 y 200 palabras por post
3. **Incluí un aprendizaje claro** — ¿qué puede sacarle el lector?
4. **Respondé cada comentario** — El algoritmo premia la interacción
5. **No tengas miedo de compartir errores** — Los posts de "lo que me salió mal" tienen 3x más engagement que los de éxito

### Ideas de contenido para las primeras 10 publicaciones

| #   | Idea                                                     |
| :-- | :------------------------------------------------------- |
| 1   | "Mi primer proyecto Django: qué aprendí"                 |
| 2   | "3 errores que cometí con Git y cómo los resolví"        |
| 3   | "¿Qué es un ORM? Lo explico como si tuvieras 5 años"     |
| 4   | "Hoy entendí la diferencia entre PUT y PATCH"            |
| 5   | "Mi stack actual: por qué elige Python y Django"         |
| 6   | "Cómo organicé mi primer proyecto con buenas prácticas"  |
| 7   | "Un bug que me costó 3 horas (y la lección que dejó)"    |
| 8   | "¿Por qué los tests son tan importantes? Mi experiencia" |
| 9   | "Lo que nadie me dijo sobre trabajar con bases de datos" |
| 10  | "Mi ruta de aprendizaje: de cero a mi primer CRUD"       |

> 🚀 **Efecto compuesto:** 10 publicaciones técnicas en 3 meses posicionan tu perfil en búsquedas de reclutadores **sin postular a una sola oferta**.

## Comenzar a postular con estrategia

### Las 5 postulaciones semanales

> **Las postulaciones personalizadas tienen una tasa de respuesta del 17% vs el 3% de las genéricas.** _(Jobscan, 2025)_

**Rutina de postulación (repetir cada semana):**

```
┌──────────────────────────────────────────────────────────────────┐
│              CICLO DE POSTULACIÓN SEMANAL                        │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. Buscar 10 ofertas alineadas con tu objetivo                  │
│                                                                  │
│  2. Filtrar las 5 más relevantes                                 │
│                                                                  │
│  3. Para cada una:                                               │
│     → Leer la descripción completa                               │
│     → Adaptar el titular del CV al nombre del rol                │
│     → Verificar que las keywords del aviso estén en el CV        │
│     → Personalizar 2-3 líneas del perfil profesional             │
│     → Verificar que el GitHub esté actualizado                   │
│     → Enviar                                                     │
│                                                                  │
│  4. Registrar en tu planilla de seguimiento                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### Plataformas recomendadas para buscar

| Tipo              | Plataformas principales                   |
| :---------------- | :---------------------------------------- |
| Empleo formal     | LinkedIn Jobs, Get on Board, Computrabajo |
| Freelance         | Workana, Upwork                           |
| Startups          | Wellfound (ex AngelList)                  |
| Prácticas/Juniors | FirstJob, Laborum, Bumeran                |
| Sector público    | empleospublicos.cl                        |

## 🎯 Hitos de Progreso — ¿Cómo sé que voy bien?

### Primer mes

- [ ] CV optimizado y en PDF
- [ ] LinkedIn completo con "Open to Work" activo
- [ ] GitHub con al menos 3 repos con README
- [ ] 4-8 publicaciones en LinkedIn

- [ ] Planilla de seguimiento activa

### Segundo mes

- [ ] 50+ conexiones relevantes en LinkedIn
- [ ] Al menos 1 respuesta positiva de reclutador
- [ ] 8-16 publicaciones acumuladas
- [ ] Perfil apareciendo en +30 búsquedas/semana en LinkedIn

- [ ] Perfil LinkedIn con tráfico constante (+50 visitas/semana)
- [ ] 1-3 procesos de selección activos

- [ ] Red de contactos de +100 personas relevantes

## ⚠️ Errores Comunes que Debes Evitar

| #   | Error                                  | Consecuencia                                         |
| :-- | :------------------------------------- | :--------------------------------------------------- |
| 1   | Enviar el mismo CV a todas las ofertas | ATS lo descarta por falta de keywords específicas    |
| 2   | No tener GitHub activo                 | El reclutador asume que no programas fuera del curso |
| 3   | LinkedIn vacío o sin foto              | Invisible para reclutadores que buscan candidatos    |
| 4   | Postular 50 ofertas sin personalizar   | 3% de tasa de respuesta vs 17% si personalizas       |
| 5   | No hacer seguimiento                   | Oportunidades que se pierden por no insistir         |
| 6   | Esperar a "estar listo" para postular  | Nunca vas a sentirte 100% listo; postulá ahora       |
| 7   | No publicar contenido en LinkedIn      | Estás en el 99% invisible de la plataforma           |
| 8   | Mentir en el CV o en entrevistas       | Descalificación inmediata y daño a tu reputación     |
