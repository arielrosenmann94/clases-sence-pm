# 🧭 1. El Mapa: Rutas para Entrar a la Industria Digital

---

Antes de hablar de CV o LinkedIn, lo más importante es entender que **no hay un solo camino**. La industria digital ofrece múltiples rutas de acceso, y cada una tiene su propio recorrido.

```
┌──────────────────────────────────────────────────────────┐
│              RUTAS DE EMPLEABILIDAD DIGITAL              │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  🏢 RELACIÓN DE DEPENDENCIA                              │
│     Empresa → Contrato → Sueldo fijo                     │
│     Perfil: Jr, Mid, Sr, Tech Lead                       │
│     Tiempo de ingreso: 1 a 4 meses                       │
│                                                          │
│  🌐 TRABAJO REMOTO INTERNACIONAL                         │
│     Salary en USD · Contratos por proyecto               │
│     Plataformas: Toptal, Arc.dev, Remote.com             │
│     Perfil: Semi-senior en adelante                      │
│                                                          │
│  🧑‍💻 FREELANCE / AUTOEMPLEO                               │
│     Por hora o por proyecto                              │
│     Plataformas: Workana, Upwork, Fiverr Pro             │
│     Tiempo de ingreso: inmediato, escalado               │
│                                                          │
│  🚀 STARTUP / EARLY-STAGE                                │
│     Menor sueldo inicial · Mayor responsabilidad         │
│     Acceso acelerado a cargos superiores                 │
│     Equity o stock options en algunos casos              │
│                                                          │
│  🏦 CONSULTORA TI                                        │
│     Rotan a distintos clientes                           │
│     Exposición alta a diferentes industrias              │
│     Formación técnica interna                            │
└──────────────────────────────────────────────────────────┘
```

> 💡 **Dato de industria:** El **42% de los empleos TI publicados en Latinoamérica son remotos o híbridos**, y el 27% acepta candidatos de cualquier país de la región. _(Bumeran Insights, 2026)_

### ¿Por dónde empezar si soy Junior?

La ruta más efectiva para profesionales en formación en 2026 es:

1. **Prácticas / Internship / Traineeship** — acceso sin experiencia previa
2. **Startups locales** — menor competencia, más tolerancia a perfiles en formación
3. **Grandes empresas** — programas formales de ingreso junior (ej: licitaciones internas, traineeship corporativo)
4. **Pymes tecnológicas** — equipos pequeños donde el junior tiene visibilidad real desde el día uno
5. **Sector público** — SEMAT, ChileAtiende, MINCIENCIA, Subsecretarías con áreas TI; ingreso por concurso vía [www.empleospublicos.cl](https://www.empleospublicos.cl/)
6. **Consultoras TI** — programas de ingreso para juniors frecuentes
7. **Freelance simple** — pequeñas tareas en Workana para generar historial

> 💡 **¿Sabías que el Estado de Chile emplea más de 7.000 profesionales TI?** El sector público es una puerta de entrada estable con contrato, horarios claros y posibilidad de conciliar estudio y trabajo. _(Servicio Civil Chile, 2025)_

# 📄 2. El Currículum Vitae en 2026: lo que Funciona y lo que no

---

## El número que cambia todo

> **El 75% de los CVs enviados son rechazados por un algoritmo antes de llegar a un humano.**
> _(Jobscan Research, 2026)_

Ese algoritmo se llama **ATS (Applicant Tracking System)**. Es un software que escanea tu CV antes de que ningún reclutador lo lea, buscando palabras clave que coincidan con el puesto. Si no aparecen esas palabras, tu CV queda fuera automáticamente.

**Lo que el ATS busca y filtra:**

| Elemento                                                | Impacto en ATS         |
| :------------------------------------------------------ | :--------------------- |
| Palabras clave del anuncio en el CV                     | ✅ +60% de visibilidad |
| Formato PDF con tipografía limpia                       | ✅ Compatible          |
| Tablas, columnas, cuadros de texto                      | ❌ No legibles por ATS |
| Fotografía, íconos gráficos complejos                   | ❌ Ignorados           |
| Títulos de sección estándar (Experiencia, Educación)    | ✅ Requeridos          |
| Títulos creativos ("Mi camino" en vez de "Experiencia") | ❌ No reconocidos      |

---

## Estructura del CV para TI en 2026

Un CV de desarrollador o perfil técnico no funciona igual que uno de otra industria. Estos son los bloques que no pueden faltar:

### 🔷 Encabezado

- Nombre completo
- Título del rol que buscás (ej: **"Backend Developer · Python · Django"**)
- Email profesional · GitHub URL · LinkedIn URL · Ciudad/País + disponibilidad remota

> ⚠️ **El 68% de los reclutadores TI verifican el GitHub antes de la primera entrevista.** _(Stack Overflow Developer Survey, 2025)_

---

### 🔷 Perfil Profesional (3-5 líneas)

Párrafo en primera persona. No describas lo que eres, describe lo que aportás.

❌ _"Soy una persona proactiva con muchas ganas de aprender"_
✅ _"Desarrollador backend con experiencia en Django y PostgreSQL. Foco en arquitectura limpia y mantenibilidad del código. Disponible para proyectos remotos."_

---

### 🔷 Tech Stack (Hard Skills)

Organizado por categorías, no como lista mezclada:

```
Lenguajes:    Python · JavaScript · SQL
Frameworks:   Django · Django REST Framework · React (básico)
Bases de datos: PostgreSQL · MySQL · SQLite
DevOps:       Git · GitHub · Docker (básico)
Metodologías: Scrum · TDD básico
```

---

### 🔷 Experiencia Laboral

Formato recomendado por reclutadores TI en 2026:

```
Desarrollador Backend Junior | Empresa Ficticia S.A. | Mar 2025 – Presente
Contexto: equipo de 5 personas, producto SaaS para gestión logística.

• Herramientas: Django, DRF, PostgreSQL, GitHub Actions
• Logros:
  → Reduje el tiempo de respuesta de 3 endpoints críticos en un 40%
     implementando select_related y caché con Redis.
  → Implementé sistema de migraciones versionadas que eliminó los
     conflictos de esquema en el equipo de 3 desarrolladores.
• Responsabilidades:
  → Mantención de modelos y migraciones de base de datos.
  → Code review de PRs de otros developers del equipo.
```

> 💡 **Regla de las métricas:** Los CVs con al menos **3 logros cuantificados tienen un 40% más de probabilidad de avanzar a entrevista.** _(Ladders Research, 2026)_

# 🔵 3. LinkedIn: el Perfil que Atrae Reclutadores

---

> **LinkedIn tiene más de 1.200 millones de usuarios en 2026. El 87% de los reclutadores usa LinkedIn como primer canal de búsqueda de candidatos.** _(LinkedIn Talent Solutions, 2026)_

Pero no alcanza con estar. El perfil tiene que estar **optimizado**.

---

## Las 8 secciones que definen tu visibilidad

### Sección 1 — Foto de perfil

- Fondo neutro o corporativo
- Rostro visible y expresión profesional
- **Los perfiles con foto reciben 21 veces más visitas que los que no la tienen** _(LinkedIn, 2026)_

### Sección 2 — Banner (portada)

- El 90% de los usuarios lo deja en blanco → oportunidad de diferenciación
- Usá una imagen con tu stack técnico o rol
- Herramienta: Canva (búsqueda "LinkedIn banner developer")

### Sección 3 — Titular (Headline)

Lo que aparece bajo tu nombre. Es lo segundo que lee el reclutador.

❌ "Estudiante de programación"
✅ "Backend Developer · Python · Django · Open to Work"

> **Los perfiles con titular específico reciben 40% más apariciones en búsquedas booleanas.** _(LinkedIn Algorithm Report, 2026)_

### Sección 4 — Resumen (Acerca de)

- Entre 200 y 300 palabras
- Primera persona
- Incluir palabras clave del rol (el algoritmo las indexa)
- Finalizar con llamado a la acción: "Abierto a oportunidades remotas, contáctame en..."

### Sección 5 — Experiencia

- Misma lógica que el CV: empresa, cargo, período, logros cuantificados
- Agregar las tecnologías usadas en cada rol

### Sección 6 — Aptitudes (Skills)

- LinkedIn indexa hasta 50 aptitudes
- Las primeras 3 son las más visibles → pones las más importantes
- Pedí validaciones (endorsements) a compañeros y mentores
- Los perfiles con +5 aptitudes validadas aparecen **17 veces más** en búsquedas _(LinkedIn, 2026)_

### Sección 7 — Recomendaciones

- Son referencias en texto de personas que trabajaron contigo
- Una recomendación de ex-compañero o mentor vale mucho en procesos de selección
- 2 o 3 recomendaciones bien escritas dan confianza al reclutador

### Sección 8 — Modo "Open to Work"

- Activa el marco verde en tu foto
- Configura qué tipo de roles buscas y si es remoto/presencial
- Los perfiles con "Open to Work" activado reciben **2,5 veces más mensajes** de reclutadores _(LinkedIn, 2026)_

## 📣 La Oportunidad Escondida: Crear Contenido en LinkedIn

> **Solo el 1% de los usuarios de LinkedIn crea contenido de forma activa, pero ese 1% genera el 100% de las 9.000 millones de impresiones semanales de la plataforma.** _(Forbes / LinkedIn Internal Data, 2025)_

Esto significa una cosa para tú: **si publicas, te ven**. La competencia es mínima comparada con el alcance potencial.

```
📊 Realidad del contenido en LinkedIn (2025)

Usuarios totales:          1.200 millones
Usuarios que CONSUMEN:     ~99%  ← ven pero no publican
Usuarios que CREAN:         ~1%  ← los que son encontrados

→ Publicar 1 vez por semana ya te pone en el 1% de visibilidad
→ El video crece al doble de velocidad que cualquier otro formato
→ El 82% de los tomadores de decisión B2B son influenciados
   por contenido de creadores de LinkedIn _(Buffer, 2025)_
```

**¿Qué publicar si eres un perfil técnico Junior?**

- ✅ Lo que aprendiste esta semana ("Hoy entendí cómo funciona ORM en Django")
- ✅ Un error que cometiste y cómo lo resolviste
- ✅ El proceso de un proyecto personal paso a paso
- ✅ Una herramienta nueva que descubriste
- ❌ Opiniones políticas o contenido no relacionado con TI

> 🚀 **Efecto compuesto:** 10 publicaciones técnicas en 3 meses posicionan tu perfil en búsquedas de reclutadores **sin postular a una sola oferta**.

## 🔍 Búsqueda Booleana: cómo te encuentran los reclutadores

Los reclutadores no buscan candidatos escribiendo nombres. Usan **operadores booleanos** (AND, OR, NOT, "") en la barra de búsqueda de LinkedIn para filtrar miles de perfiles en segundos:

```text
🔎 LinkedIn Search Bar Simulation
┌─────────────────────────────────────────────────────────────────────────────┐
│ ("backend developer" OR "desarrollador backend") AND (Python OR Django)     │
│ AND (Junior OR "entry level") NOT Senior NOT Lead                           │
└─────────────────────────────────────────────────────────────────────────────┘
```

> **¡El algoritmo es ciego!**
> Si tu perfil no contiene los términos exactos de la búsqueda, **no aparecerás** en los resultados, sin importar qué tan buen programador seas.

**¿Qué debes hacer hoy?**

1.  Lee a lo menos 10 avisos de trabajo que te apasionen.
2.  Extrae las palabras técnicas más repetidas.
3.  Asegúrate de que aparezcan en tu **Titular**, **Acerca de** y **Aptitudes**.

## Cómo postular de forma efectiva

**No envíes el mismo CV a 50 empresas.** La estrategia más efectiva en 2026:

```
CALIDAD > CANTIDAD

→ 5 postulaciones personalizadas por semana
   > 50 postulaciones genéricas por semana

Los datos lo confirman: las postulaciones personalizadas
tienen una tasa de respuesta del 17% vs el 3% de las genéricas.
(Jobscan, 2026)
```

**Checklist antes de cada postulación:**

- [ ] Leer la descripción del puesto completa
- [ ] Adaptar el titular del CV al nombre exacto del rol
- [ ] Verificar que las palabras clave del anuncio estén en el CV
- [ ] Personalizar 2-3 líneas del perfil profesional del CV
- [ ] Verificar que el GitHub esté actualizado y visible

### El correo de postulación espontánea

```
¿Habría espacio para una conversación de 15 minutos?

Saludos,
[Tu nombre]
```

# 🎯 5. El Proceso de Selección: Etapa a Etapa

---

En TI, los procesos de selección suelen tener entre 3 y 6 etapas. Cada una tiene un objetivo distinto y requiere preparación diferente.

```
┌──────────────────────────────────────────────────────────┐
│           PROCESO DE SELECCIÓN EN TI (2026)              │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  [1] REVISIÓN DE CV / ATS                                │
│       ↓  ~75% se filtra aquí                             │
│  [2] ENTREVISTA CON RECLUTADOR (RRHH)                    │
│       ↓  ~40% pasa esta etapa                            │
│  [3] ENTREVISTA TÉCNICA (Líder o Dev Senior)             │
│       ↓  ~50% pasa esta etapa                            │
│  [4] CHALLENGE TÉCNICO / TAKE-HOME / LIVE CODING         │
│       ↓  ~60% entrega, ~50% aprueba el resultado         │
│  [5] ENTREVISTA FINAL (Hiring Manager / CEO en startup)  │
│       ↓  ~70% de los que llegan aquí reciben oferta      │
│  [6] OFERTA Y NEGOCIACIÓN                                │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

---

## 5.1 — Revisión de CV (ATS + Reclutador)

Ya lo vimos: el 75% se filtra automáticamente. Lo que el reclutador mira en los primeros 6 segundos:

- **Titular del rol** (¿coincide con lo que buscan?)
- **Empresa anterior** (¿es conocida o relevante?)
- **Tech stack visible** (¿aparecen las tecnologías pedidas?)
- **GitHub/Portfolio** (¿está activo?)

> **Los reclutadores dedican en promedio 6,2 segundos a la primera lectura de un CV.** _(The Ladders Eye-Tracking Study, 2026)_

---

## 5.2 — Entrevista con Reclutador (RRHH)

**Objetivo:** verificar fit cultural, expectativas salariales y disponibilidad.

Las preguntas más comunes:

- "Contame de tú" — quieres al elevator pitch de 60 segundos
- "¿Por qué este puesto / esta empresa?"
- "¿Cuál es tu expectativa salarial?"
- "¿Cuándo podrías incorporarte?"
- "¿Tienes experiencia trabajando en equipo?"

**Tip crítico para la expectativa salarial:**

> No des un número único. Dá un rango basado en tu investigación de mercado.
> _"Basándome en el mercado para este rol con mi experiencia, estoy buscando entre $X y $Y. ¿Está dentro del rango del puesto?"_

---

## 5.3 — Entrevista Técnica

**Objetivo:** validar que los conocimientos del CV son reales.

Puede ser con el tech lead, un senior developer o el CTO de una startup.

**Formato más común:**

- Preguntas conceptuales sobre el stack (¿cómo funciona X? ¿cuál es la diferencia entre A y B?)
- Revisión de código del candidato (GitHub o proyectos)
- Discusión de decisiones técnicas tomadas en proyectos anteriores

**Lo que se evalúa:**

- Profundidad del conocimiento (no memoria, sino comprensión)
- Capacidad de explicar conceptos
- Cómo razonás cuando no sabes algo
- Si pedís ayuda o improvisás sin base

**Respuesta modelo cuando no sabes:**

> _"No tengo certeza de la respuesta exacta, pero lo razonaría así: [razonamiento]. Lo que sí sé es que [lo que sí sabes relacionado]."_
> Eso es mucho mejor que inventar.

---

## 5.4 — Challenge Técnico

**Objetivo:** ver cómo construyes código real, sin presión de entrevistador presente.

Existen dos formatos:

### Take-home challenge

Se entrega un enunciado y el candidato tiene 24-72 horas para resolverlo.

**Lo que se evalúa en el código del challenge:**
| Criterio | Qué buscan |
| :--- | :--- |
| Funcionalidad | ¿El sistema hace lo que el enunciado pedía? |
| Calidad del código | ¿Es legible, con nombres descriptivos? |
| Estructura del proyecto | ¿Tiene sentido la organización de archivos? |
| README | ¿Se puede correr sin ayuda? |
| Decisiones técnicas | ¿Las puede explicar en la entrevista siguiente? |
| Tests (si aplica) | ¿Tiene al menos pruebas básicas? |

### Live coding

Resolución en tiempo real en pantalla compartida con el entrevistador.

**Tips para live coding:**

- Pensá en voz alta siempre. El proceso importa tanto como el resultado
- Antes de escribir código, describe cómo lo vas a encarar
- Preguntá si tienes dudas del enunciado (es válido y positivo)
- Si te bloqueás, decilo: _"Me trabé aquí, voy a intentar otro ángulo"_

> **El 63% de las empresas TI usa algún tipo de challenge técnico en su proceso de selección.** _(HackerRank Developer Skills Report, 2026)_

---

## 5.5 — Entrevista Final

**Objetivo:** fit con la cultura, alineación de expectativas, negociación.

Suele ser con el hiring manager, el gerente de área o el fundador en startups.

Las preguntas más frecuentes en esta etapa:

- "¿Por qué quieres trabajar específicamente aquí?"
- "¿Dónde te ves en 2 años?"
- "¿Cómo manejas los conflictos dentro de un equipo?"
- "¿Tienes preguntas para nosotros?"

**Esa última pregunta es clave.** Las respuestas que más impactan:

- _"¿Cómo es el proceso de onboarding para perfiles nuevos?"_
- _"¿Cuáles son los principales desafíos técnicos del equipo hoy?"_
- _"¿Cómo miden el éxito de alguien en este rol a los 90 días?"_

# ⚠️ 7. Razones Más Comunes de Despido en la Industria TI

---

Conocer qué lleva a un despido es tan importante como saber conseguir el trabajo. La mayoría de los despidos en TI **no son por falta de habilidades técnicas**.

> **El 89% de los despidos en la industria tecnológica están relacionados con soft skills, actitud o comportamiento, no con habilidades técnicas.** _(Leadership IQ Study, 2026)_

---

| #      | Razón                                     | Qué significa en la práctica                                     |
| :----- | :---------------------------------------- | :--------------------------------------------------------------- |
| **1**  | **No cumplir con plazos y compromisos**   | Decir "sí" a todo sin avisar cuando hay problemas                |
| **2**  | **Mala comunicación**                     | No avisar bloqueos, no documentar, respuestas vagas              |
| **3**  | **Resistencia al feedback**               | Recibir una corrección de código y reaccionar a la defensiva     |
| **4**  | **Código sin mantenibilidad**             | Código que nadie más puede entender o modificar                  |
| **5**  | **No adaptarse al equipo**                | Trabajar en silo, no hacer code review, no colaborar             |
| **6**  | **Seguridad y confidencialidad**          | Exponer credenciales, subir secrets a GitHub, ignorar políticas  |
| **7**  | **Estancamiento técnico**                 | No actualizar conocimientos en un sector que cambia cada 6 meses |
| **8**  | **Conflictos interpersonales reiterados** | Problemas de convivencia que afectan al equipo                   |
| **9**  | **No documentar el trabajo**              | El conocimiento que no se escribe se pierde con la persona       |
| **10** | **Fraude en el proceso de selección**     | Mentiras en el CV o proyectos que no son propios                 |

---

## El error más frecuente en los primeros 3 meses

> **El 46% de los profesionales que son despedidos en el primer año fallan por razones actitudinales en los primeros 90 días.** _(Society for Human Resource Management, 2026)_

Los primeros 90 días son el período donde se establece la reputación dentro del equipo. Las reglas no escritas de ese período:

- Preguntá (mucho, pero con criterio — primero intentá tú)
- Entregá cosas pequeñas pero completas
- Documentá todo lo que aprendes
- No prometás lo que no puedes cumplir
- Participá en reuniones aunque no tengas mucho para decir
